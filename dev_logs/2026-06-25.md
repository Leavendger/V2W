### P10a — AI 会议总结

详情页加「✨ AI 总结」按钮，**手动点击才生成（默认不总结）**，产出摘要 + 行动项 + 关键词。

#### 技术实现

- **统一 OpenAI 兼容客户端**：一套 `summarizer.py` 通吃 DeepSeek / GLM / 通义 / OpenAI / 本地 Ollama —— provider 只是 `{base_url, model, api_key}` 三元组，靠预设表驱动
- **配置驱动多 provider**：`llm_providers.json` 预设表（入 git 示例，可自由增删）+ `config.current_llm_provider()`，换厂商改 `.env` 零代码改动，默认 DeepSeek
- **复用 worker 队列**：新增 `('summarize', id)` 任务，与转写/重识别同构，不阻塞请求
- **长文本 map-reduce**：逐字稿超 6000 字分段提取要点再合并，避免长会议截断或质量下降
- **结构化 + 容错**：`response_format: json_object` + 多级 JSON 解析兜底（去代码块 / 提取 `{...}` / 整段兜底）

#### 交互流程

1. 默认不总结，已完成文件详情页显示「✨ 生成 AI 总结」按钮
2. 点击 → confirm → POST `/summarize`（创建 Summary.status=summarizing + 入队）→ reload 显示 loading
3. 前端轮询 `/api/file/<id>/summary`，done/failed 自动 reload
4. 完成后展示 摘要 / 行动项(☐) / 关键词 chips + provider·model 元信息 + 「重新生成」
5. 未配置 API key → `summary_enabled=False`，按钮不渲染；POST 返回友好错误
