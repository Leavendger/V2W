### 关键实现要点

1. **搜索逻辑收拢在 `doSearch()`**：防抖 300ms → fetch API → 命中段加 `.search-hit`、`↑/↓`+回车循环导航、`Esc`/`✕` 清除、计数 `当前/总数`
2. **高亮由前端完成**（`highlight()` + `esc()` 转义）：规避 Python(Unicode 码点) 与 JS(UTF-16 码元) 在 emoji 文本上的索引错位
3. **与播放器共存**：新增 `searchActive` 标志，搜索激活期间 `timeupdate` 的 `scrollIntoView` 被守卫跳过（仍保留 `.active` 高亮），避免搜索定位与播放跟随互相抢占滚动
4. **P7 深链预留**：`detail.html` 读取 URL `?q=` 自动填入并触发搜索，P7 结果页可直接深链定位

## 关键决策记录

1. **LIKE 而非 FTS5**：单用户、单文件数百段量级，LIKE 毫秒级且中文子串匹配天然友好；FTS5 需纠结中文 tokenizer，得不偿失，留作升级路径
2. **高亮前端化**：后端只返回命中段落、不返回 `match_offsets`，消除跨语言索引差异风险

## 验证结果

后端 API 全绿（`venv/bin/python app.py` + curl）：

| 用例 | 结果 |
|------|------|
| 中文 `q=用户`（file 1） | ✅ 命中 2 条（段 1、2） |
| 通配符 `q=25%`（file 1） | ✅ 仅匹配字面"25%"1 条，**`%` 转义生效** |
| 空查询 `q=` | ✅ `{total:0, hits:[]}` |
| 无命中 `q=zzzzz` | ✅ `{total:0}` |
| 大文件 `q=平凡`（file 2，234 段） | ✅ 命中 15 条，按 segment_index 升序 |

`py_compile` 通过；IDE 对 `detail.html` 无诊断。
