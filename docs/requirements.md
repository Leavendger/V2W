# V2W — 产品需求文档 (PRD)

> 面向产品经理的 AI 会议助手 · MVP 版本

## 1. 产品概述

V2W (Voice to Words) 是一款 Web 端的会议记录工具。用户上传会议录音/视频，系统自动将语音转为带时间戳的文字，用户可在网页中播放音视频并点击文字段落快速跳转到对应位置。

### 目标用户
- 产品经理（首要用户）
- 需要整理会议记录、访谈记录的职场人士

### 核心价值
- 上传音视频 → 自动转写 → 快速定位关键内容
- 本地处理，数据不外传，保障会议隐私

## 2. MVP 功能范围

### 2.1 文件上传
- 支持拖拽或点击上传
- 支持常见音视频格式：mp4, mov, avi, wmv, webm, mp3, wav, m4a, flac
- 单文件上传，大小限制 500MB
- 上传后显示进度，完成后跳转首页

### 2.2 文件库
- 首页卡片网格展示所有已上传文件
- 每张卡片显示：文件名、类型图标、时长、状态标签
- 状态流转：🟡 已上传 → 🔵 转写中 → 🟢 已完成（🔴 失败）
- 支持点击卡片进入详情，支持删除文件

### 2.3 音视频预览
- 详情页内嵌 HTML5 播放器
- 支持视频画面预览 / 纯音频波形播放
- 标准播放控件（播放/暂停/进度条/音量）

### 2.4 语音转文字
- 后台自动转写，同一时间只处理 1 个文件（排队机制）
- 提取词级时间戳，聚合为句子级段落
- 转写语言：自动检测（支持中英混合）

### 2.5 转写结果预览
- 文字按时间戳分段列表展示
- 每条段落显示：开始时间标签 + 文字内容
- **点击文字段落 → 播放器跳转到对应时间开始播放**
- 当前播放到的段落自动高亮

### 2.6 数据持久化
- 关闭浏览器再打开，历史文件和转写结果保留
- 刷新页面不丢失数据

### 2.7 全文搜索（下一迭代，已立项设计）

> 详细设计见 [docs/search-design.md](search-design.md)，分两阶段推进：

- **阶段一（执行计划 P6）· 单文件内搜索**：在详情页内输入关键词，高亮所有命中段落，`↑/↓` 逐条导航，点击命中段跳转播放
- **阶段二（执行计划 P7）· 全局搜索**：导航栏搜索框跨文件检索，**同时匹配项目名（文件名）与转写文字**，结果分「全部 / 项目名 / 转文字」三类 Tab，点击直达详情页对应位置

## 3. 不在 MVP 范围（后续迭代）

以下功能记录在 README.md 中，MVP 暂不实现：
- 说话人区分
- AI 自动总结
- 关键词提取
- 待办事项提取
- 章节自动划分
- 导出功能（Word / SRT / Markdown）
- 多语言翻译
- 实时录音转写

## 4. 用户故事

| 编号 | 作为 | 我想要 | 以便 |
|------|------|--------|------|
| US-1 | 用户 | 上传会议录音文件 | 让系统自动转写 |
| US-2 | 用户 | 看到所有已上传的文件列表 | 管理和回顾历史记录 |
| US-3 | 用户 | 在网页中播放音视频 | 无需切换播放器软件 |
| US-4 | 用户 | 查看转写出的文字和对应时间 | 快速浏览会议内容 |
| US-5 | 用户 | 点击某段文字跳转到对应播放位置 | 快速定位到感兴趣的片段 |
| US-6 | 用户 | 知道转写是否完成 | 合理安排等待时间 |

## 5. 验收标准

- [ ] 可上传 mp4/mp3/wav 文件并存储
- [ ] 首页列出自上传以来的所有文件
- [ ] 上传后自动开始转写，无需手动触发
- [ ] 转写完成后详情页显示带时间戳的文字
- [ ] 点击文字段落，播放器跳转到对应时间
- [ ] 刷新页面后数据依然存在
- [ ] UI 为淡蓝色调，简洁直观
