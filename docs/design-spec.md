# V2W — UI 设计规范

> MVP 版本界面设计标准

## 1. 设计原则

- **简洁直观**：减少认知负担，核心操作不超过 2 次点击
- **信息层次**：文件列表 → 详情播放 → 转写文字，层层递进
- **状态可见**：上传进度、转写状态始终可见

## 2. 色彩系统

### 主色调（淡蓝）

| 用途 | 色值 | 预览 |
|------|------|------|
| 页面背景 | `#f0f6fc` | 极淡蓝灰 |
| 卡片背景 | `#ffffff` | 白色 |
| 主按钮 | `#1976d2` | 深蓝 |
| 主按钮悬停 | `#1565c0` | 更深蓝 |
| 导航栏 | `#ffffff` + 底部阴影 | 白色导航 |
| 强调文字 | `#1976d2` | 深蓝 |
| 已完成状态 | `#4caf50` | 绿色 |
| 处理中状态 | `#1976d2` | 蓝色 + 动画 |
| 失败状态 | `#f44336` | 红色 |
| 文字主色 | `#212121` | 近黑色 |
| 文字次要 | `#757575` | 灰色 |

### CSS 变量定义

```css

```

## 4. 组件规范

### 文件卡片

```
宽度: 200px，自适应 grid
圆角: 8px
阴影: var(--shadow)
内容: 文件图标 + 文件名(截断) + 时长 + 状态标签
悬停: 阴影加深，轻微上移 translateY(-2px)
```

### 状态标签

| 状态 | 样式 | 文字 |
|------|------|------|
| uploaded | 黄色背景 `#fff3e0`，橙色文字 `#e65100` | 排队中 |
| processing | 蓝色背景 `#e3f2fd`，蓝色文字 `#1976d2` + 脉冲动画 | 转写中… |
| completed | 绿色背景 `#e8f5e9`，绿色文字 `#2e7d32` | 已完成 |
| failed | 红色背景 `#ffebee`，红色文字 `#c62828` | 失败 |

### 上传按钮

```
背景: var(--primary)
文字: 白色
圆角: 24px（胶囊形）
图标: + 号
悬停: var(--primary-hover)
```

### 转写段落

```
布局: 左侧时间标签 + 右侧文字
时间标签: 等宽字体，次要色
文字: 主色，行高 1.6
当前播放段落: 左侧蓝色竖线 + 淡蓝背景
点击交互: cursor:pointer，悬停浅灰背景
```

### 搜索栏（迭代 P6）

> 详见 [search-design.md](search-design.md)，置于转写区标题「📝 转写内容」右侧。

```
🔍 [ 输入框____________ ] [↑ 上一个] [↓ 下一个]   3/12 处命中   [✕]
```

- 输入框：圆角，聚焦时淡蓝边框（`--primary`），防抖 300ms 触发
- 命中段落：黄底 `#fff3cd`
- 当前定位的命中段：深黄底 `#ffe082` + 蓝色描边
- 关键词：`<mark>` 黄底标记，文字保持主色
- 与播放高亮共存：播放跟随用蓝系（竖线 + 淡蓝背景），搜索用黄系，互不覆盖
- 搜索激活期间，暂停播放的自动滚动跟随（避免与命中定位互相抢占）

### 全局搜索结果页（迭代 P7）

> 详见 [search-design.md](search-design.md) 第 8 节。入口为导航栏搜索框。

```
🔍 [ 关键词________ ]              [搜索]
  [ 全部 N ] [ 项目名 N ] [ 转文字 N ]
```

- **Tab 栏**：底部下划线指示当前分类（`--primary`），药丸标签显示计数；切换为链接跳转
- **项目名命中条目**：文件图标 + 文件名（`<mark>` 高亮）+ 状态标签 + 时长，整条可点进详情
- **内容命中**：按文件分组卡片，文件名 + 命中数；条目为时间戳 + 高亮片段，点击深链跳转
- **「全部」分两块**：项目名命中在上、内容命中在下
- **空状态**：全无命中 / 当前分类无结果，两种提示

### 导出按钮（迭代 P8）

> 详情页头部右侧，仅 `completed` 文件显示。

- 样式：描边按钮（白底 + 蓝边 `--primary` + 蓝字），hover 反色填充为实心蓝
- 文案：「⬇ 导出 Markdown」
- 点击：`GET /file/<id>/export`，浏览器下载 `{原文件名}.md`
- 与状态标签并列于 `.detail-header-actions`

### 说话人标签（迭代 P9）

> 详见 [speaker-diarization-design.md](speaker-diarization-design.md)。仅在开启说话人分离时显示。

- 位置：段落文字前（段首小标签）
- 文案：「说话人 1 / 2 / 3」（pyannote 的 `SPEAKER_00` 映射）
- 样式：药丸标签，蓝系（与时间戳协调）
- 未识别（未开启分离 / 失败）：不显示标签，导出仍为 `[发言人]` 占位

## 5. 响应式

- **桌面端（>1024px）**：详情页左右分栏（播放器 + 转写区）
- **平板/手机（<768px）**：上下堆叠，播放器固定顶部
- **最小支持宽度**：375px

## 6. 交互规范

| 操作 | 反馈 |
|------|------|
| 上传文件 | 进度条 → 成功提示 → 返回列表 |
| 转写中 | 状态标签脉冲动画 + 自动轮询刷新 |
| 转写完成 | 状态变为绿色，详情页显示文字 |
| 点击段落 | 播放器时间跳转 + 段落高亮 + 自动播放 |
| 播放中 | 当前段落自动高亮跟随 |
| 删除文件 | 确认弹窗 → 删除 → 列表刷新 |
