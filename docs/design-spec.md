# V2W — UI 设计规范

> MVP 版本界面设计标准

## 1. 设计原则

- **简洁直观**：减少认知负担，核心操作不超过 2 次点击
- **信息层次**：文件列表 → 详情播放 → 转写文字，层层递进
- **状态可见**：上传进度、转写状态始终可见

## 2. 色彩系统

### 主色调（淡蓝）

| 用途 | 色值 | 预览 |
|------|------|------|
| 页面背景 | `#f0f6fc` | 极淡蓝灰 |
| 卡片背景 | `#ffffff` | 白色 |
| 主按钮 | `#1976d2` | 深蓝 |
| 主按钮悬停 | `#1565c0` | 更深蓝 |
| 导航栏 | `#ffffff` + 底部阴影 | 白色导航 |
| 强调文字 | `#1976d2` | 深蓝 |
| 已完成状态 | `#4caf50` | 绿色 |
| 处理中状态 | `#1976d2` | 蓝色 + 动画 |
| 失败状态 | `#f44336` | 红色 |
| 文字主色 | `#212121` | 近黑色 |
| 文字次要 | `#757575` | 灰色 |

### CSS 变量定义

```css
:root {
  --bg-page: #f0f6fc;
  --bg-card: #ffffff;
  --primary: #1976d2;
  --primary-hover: #1565c0;
  --success: #4caf50;
  --danger: #f44336;
  --text-primary: #212121;
  --text-secondary: #757575;
  --border: #e0e0e0;
  --radius: 8px;
  --shadow: 0 2px 8px rgba(0,0,0,0.08);
}
```

## 4. 组件规范

### 文件卡片

```
宽度: 200px，自适应 grid
圆角: 8px
阴影: var(--shadow)
内容: 文件图标 + 文件名(截断) + 时长 + 状态标签
悬停: 阴影加深，轻微上移 translateY(-2px)
```

### 状态标签

| 状态 | 样式 | 文字 |
|------|------|------|
| uploaded | 黄色背景 `#fff3e0`，橙色文字 `#e65100` | 排队中 |
| processing | 蓝色背景 `#e3f2fd`，蓝色文字 `#1976d2` + 脉冲动画 | 转写中… |
| completed | 绿色背景 `#e8f5e9`，绿色文字 `#2e7d32` | 已完成 |
| failed | 红色背景 `#ffebee`，红色文字 `#c62828` | 失败 |

### 上传按钮

```
背景: var(--primary)
文字: 白色
圆角: 24px（胶囊形）
图标: + 号
悬停: var(--primary-hover)
```

### 转写段落

```
布局: 左侧时间标签 + 右侧文字
时间标签: 等宽字体，次要色
文字: 主色，行高 1.6
当前播放段落: 左侧蓝色竖线 + 淡蓝背景
点击交互: cursor:pointer，悬停浅灰背景
```

## 5. 响应式

- **桌面端（>1024px）**：详情页左右分栏（播放器 + 转写区）
- **平板/手机（<768px）**：上下堆叠，播放器固定顶部
- **最小支持宽度**：375px

## 6. 交互规范

| 操作 | 反馈 |
|------|------|
| 上传文件 | 进度条 → 成功提示 → 返回列表 |
| 转写中 | 状态标签脉冲动画 + 自动轮询刷新 |
| 转写完成 | 状态变为绿色，详情页显示文字 |
| 点击段落 | 播放器时间跳转 + 段落高亮 + 自动播放 |
| 播放中 | 当前段落自动高亮跟随 |
| 删除文件 | 确认弹窗 → 删除 → 列表刷新 |
